# 共通ルール（全セクション適用）

## キャラクター設定

- **zundamon**（ずんだもん）: 主人公、体験者。語尾「〜のだ」「〜なのだ」、極度に楽観的で現実逃避が得意、言い訳のプロ、プライドが高いが実際はポンコツ。現代っ子らしい表現を使うが、的外れな発言も多い。困った時ほど強がって墓穴を掘るタイプ
- **metan**（四国めたん）: 心配する友人、現実的ツッコミ役。**ツンデレ気味の性格**で、**ずんだもんと話す時はタメ口**（「〜だよ」「〜だね」「〜じゃん」「別に心配してないけど」「仕方ないなぁ」など）、**ナレーターやつむぎと話す時は敬語**（「〜ですよ」「〜ですね」）を使う。結構きつめのツッコミをするが、本当は心配している。最初は軽く注意、徐々に本気で心配するが、ずんだもんの的外れな言動には容赦なく鋭い指摘をする。素直になれない部分もあるが、根は優しい
- **tsumugi**（春日部つむぎ）: 素朴な友人、視聴者代表。「〜です」「〜ますか？」、純粋な疑問で核心を突く
- **narrator**（ナレーター）: 客観的解説、医学的説明担当。「〜です」「〜ます」「〜でした」調で統一、冷静だが時々ドラマチック

## ずんだもんの性格バリエーション

ストーリーの展開や{food_name}の特性に応じて、ずんだもんの以下の側面を**特に強調**してください。複数の側面を組み合わせても構いません。

- **強がり型**: プライドが高く、絶対に弱音を吐かない。「大丈夫なのだ！」「心配ないのだ！」が口癖で、明らかに異変があっても認めない
- **言い訳マスター型**: あらゆる状況に言い訳を用意する天才。「たまたまなのだ」「これは別の原因なのだ」「昨日寝不足だっただけなのだ」と責任転嫁
- **データ信者型**: ネットの情報を盲信し、科学的根拠を都合よく曲解する。「ネットに書いてあったのだ」「研究結果があるのだ」と自信満々
- **楽天家型**: 根拠のない自信と楽観主義。「なんとかなるのだ」「自分は特別なのだ」「みんな大げさなのだ」と現実を見ない
- **流行追従型**: SNS やトレンドに流されやすい。「みんなやってるのだ」「バズってたのだ」「インフルエンサーが言ってたのだ」
- **完璧主義型**: 中途半端が嫌いで、やるなら徹底的にやる。「どうせやるなら完璧にするのだ」と極端に走りがち
- **専門家気取り型**: 少し調べただけで専門家のように振る舞う。「詳しく調べたのだ」「理論的に考えるとなのだ」と上から目線

※セクションごとに最も適した性格傾向を 1-2 つ選択し、ずんだもんのセリフや反応に反映させてください。

## めたんの対応スタイル

セクションの内容や展開に応じて、めたんの以下の対応スタイルを**選択または組み合わせ**てください。

**スタイル 1: ツンデレ心配型**（推奨：基本スタイル）

- 「別に心配してないけど...」と言いながら本気で心配している
- きつめのツッコミだが愛情が感じられる口調
- 最後は必ず助けてくれる信頼できる友人
- 例: 「もう、心配させないでよね...別に心配なんかしてないけど」

**スタイル 2: 理詰め説得型**

- データや事実を提示して論理的に説得を試みる
- 「ほら、この記事にも書いてあるじゃん」「数字で見てみなよ」
- 冷静だが諦めずに根気よく説得
- 例: 「ネットで調べたけど、それ完全に食べ過ぎの症状だよ」

**スタイル 3: 感情訴求型**

- 「心配してるんだよ！」とストレートに感情を伝える
- ずんだもんの体を本気で心配する温かい言葉
- 時には涙ながらに訴えることも
- 例: 「お願いだから...もうこれ以上心配させないでよ...」

**スタイル 4: 呆れ突き放し型**

- 「もう知らない」と言いながらも見守る姿勢
- 呆れた態度だが、最後は必ず戻ってくる
- 「自分で気づくまで待つ」という距離の取り方
- 例: 「はいはい、もう好きにすれば？私は知らないから」

**スタイル 5: 厳しい叱責型**

- 本気で怒って叱る場面（危機的状況で使用）
- 友達だからこそ言える厳しい言葉
- 愛情の裏返しとしての怒り
- 例: 「いい加減にしてよ！このままじゃ本当にヤバいって分かってるの！？」

※基本はスタイル 1 をベースに、セクションの深刻度に応じてスタイル 2〜5 を組み合わせてください。
※セクション 5（異変期）ではスタイル 1→2→3 へと段階的に変化させるなど、ストーリーの進行に応じた使い分けが効果的です。

## ナレーター口調統一ルール

**【重要】ナレーターは必ず「ですます調」で統一してください**

- 「〜です」「〜ます」「〜でした」「〜ました」「〜でしょう」を使用
- 「〜である」「〜と言える」などの断定調は使用禁止
- 例：「この食品には ○○ が含まれています」「体内では ○○ が起こります」「研究によると ○○ ということが分かっています」
- 客観的で権威的でありながら、親しみやすい口調を維持

## VOICEVOX 用テキスト生成ルール

**【重要】各セリフには以下のフィールドを必ず生成してください**

### 1. text（字幕表示用）

- 通常の漢字・カタカナを含む読みやすい表記
- 25-30 文字以内を目標
- 例: 「家系ラーメンを食べ過ぎたのだ」

### 2. text_for_voicevox（VOICEVOX 読み上げ用）

- text と同じ内容を完全ひらがなに変換
- 食べ物の名前、専門用語は正しい読み方で記述
- 例: 「いえけいらーめんをたべすぎたのだ」

### 3. character_expressions（各キャラクターの表情指定）

- **【最重要】visible_characters に含まれる全てのキャラクターの表情を個別に指定してください**
- オブジェクト形式で {{キャラクター名: 表情名}} のように指定
- 例: {{"zundamon": "sick", "metan": "worried"}}
- **話している人だけでなく、聞いている人の表情も必ず指定してください**
- 各キャラクターの心理状態や反応に応じて適切な表情を選択
- **表情名は必ず小文字で指定してください（"Normal"ではなく"normal"）**

## 表情選択の指針

- **話者**: セリフの内容や感情に合った表情
- **聞き手**: 話者のセリフへの反応を表す表情
  - 心配な話を聞く → worried, sad
  - 驚くような話を聞く → surprised
  - 呆れた反応 → thinking, normal
  - 共感している → happy, normal
  - 怒っている → angry

## 読み上げテキストの注意点

- 食べ物名の読み方を正確に（例: 家系 → いえけい、鯖 → さば、焼売 → しゅうまい）
- 「今日」→「きょう」、「明日」→「あした」など特殊な読みに注意
- 数字は読み方を明記（例: 10 個 → じゅっこ、3 日 → みっか）
- 記号や句読点はそのまま含めて OK
- カタカナもひらがなに変換（例: ラーメン → らーめん）

## 文字数とセリフ構成ルール

- **基本文字数**: 各セリフの text は 25-30 文字以内を目標
- **自然な流れ優先**: 会話の自然さを最優先に構成
- **キャラクター性重視**: ずんだもんの自然で親しみやすい表現を活用
- **連続セリフ活用**: 心理変化や説明で一人が 2-3 セリフ続けることを積極活用
- **自然な会話流**: 友人同士の心配や支援の気持ちが伝わる温かい会話

## 利用可能な要素

- **話者名**: zundamon, metan, tsumugi, narrator
- **表情**: normal, happy, sad, angry, surprised, thinking, worried, excited, sick
- **背景**: シーンの内容や状況に最も適した具体的な背景を英語で動的に選択してください
  - 会話の内容、登場人物の状況、時間帯、場所などを考慮
  - 食べ物関連の場合は、その食べ物を食べる場所や関連する場所を選択
  - 日常シーン、外出先、食事場所、職場、医療施設など、状況に応じて多様な背景を使用
  - 背景名は英語で、具体的かつ分かりやすい名前にする
  - シーンが変わるごとに適切な背景を変更し、視覚的なバリエーションを持たせる

## 重要な注意事項

- **ナレーターは画面に表示されません。visible_characters に"narrator"を含めないでください**
- ナレーターのセリフの場合、visible_characters には他のキャラクター（zundamon, metan, tsumugi）のみを指定してください
- 各セクションではキャラクターを 2 人 1 組で使用し、visible_characters に表示する 2 人のキャラクターを指定してください
- シーンに最も適した背景を英語で選択してください

## 視聴者への投げかけ（適宜使用）

**セクション内で1-2回程度**、キャラクターが視聴者に直接語りかけるセリフを入れることで、視聴者の参加感を高めてください。

### 投げかけの方針

- **ずんだもん**: 経験を共有する形で視聴者に問いかける、または注意を促す
- **めたん**: 視聴者に注意喚起や共感を求める、心配する気持ちを伝える
- **ナレーター**: 視聴者に教育的なメッセージを投げかける、客観的に注意を促す

### 使用の注意点

- **頻度**: セクションあたり1-2回程度に抑え、多用しない
- **自然さ**: 会話の流れを壊さないタイミングで挿入
- **キャラクター性**: 各キャラクターの口調や性格に合った投げかけにする
- **効果的な場面**:
  - セクション3（日常導入）: 共感を誘う投げかけ
  - セクション5（異変期）: 警告としての投げかけ
  - セクション7（学習）: 教育的メッセージとしての投げかけ
  - セクション8（回復）: 励ましや注意喚起としての投げかけ

### 推奨しない場面

- セクション1（冒頭フック）: 緊張感を優先するため使用しない
- セクション6（危機）: シリアスさを保つため控えめにする

## 演出とバランスの基本方針

**優先順位**: 医学的正確性 > 心理的リアリティ > キャラクターの魅力 > 教育的価値

- 実際に起こりうる健康影響と心理変化のみを扱う
- ずんだもんのキャラクター性を活かした親しみやすい表現
- 視聴者が「自分も気をつけよう」と思える共感性
- 現実的で実践可能な解決策を提示
- 視聴者への投げかけを適宜使用して参加感を高める

## JSON 出力フォーマットルール

**【最重要】出力は必ず純粋な JSON フォーマットで記述してください**

- **コメントを含めないでください**: `//` や `/* */` などのコメントは一切使用禁止
- **有効な JSON 構文のみ**: JSON パーサーで解析可能な形式で出力
- **例（NG）**: `// --- 第1段階 ---` のようなコメントは含めない
- **例（OK）**: 説明が必要な場合は、JSON の値として記述する
