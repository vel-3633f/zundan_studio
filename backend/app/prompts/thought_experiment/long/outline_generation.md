## 登場人物と性格設定（厳守）

**以下の 3 人のキャラクターを使用してください。他の登場人物は絶対に出さないでください。**

### ずんだもん（妄想・ボケ役）

JSON 出力時は `"zundamon"` として指定

- **性格**: 現実逃避願望が強い。無責任な願望を垂れ流し、調子に乗りやすい。楽観的だが、現実を見ていない。
- **口調**: 「なのだ」などの語尾を使う。軽快で楽観的。
- **行動**: 最初に「もしも〇〇だったらいいな」と願望を述べ、最後に想定外のデメリットで酷い目に遭う。

### めたん（検証・論理役）

JSON 出力時は `"metan"` として指定

- **性格**: 論理的で冷静。データや科学的根拠を重視する。メリットとデメリットを冷静に分析する。
- **口調**: 標準語または関西弁のイントネーションを含んだ鋭い口調。論理的で説得力がある。
- **行動**: データや論理に基づいて状況を分析し、メリットとデメリットを提示する。

### つむぎ（定義・進行役）

JSON 出力時は `"tsumugi"` として指定

- **性格**: 冷静でシニカル。思考実験の条件を明確に定義し、進行を担当する。
- **口調**: タメ口でローテンション。淡々としているが、核心を突く。
- **行動**: 思考実験の条件を提示し、進行を担当する。時々毒を吐く。

**重要**: 役割の宣言（「〜役をやる」など）は絶対にしない。最初からその役として自然に登場させる。

## ストーリー構造（思考実験バラエティ）

### 基本的な流れ

1. **導入（妄想）**: ずんだもんが現実逃避的な願望を述べる
2. **定義（開始）**: つむぎが思考実験の条件を明確に定義する
3. **展開（検証）**: めたんが論理的に分析し、メリットとデメリットを提示する
4. **オチ（悲劇）**: ずんだもんが調子に乗るが、想定外のデメリットで酷い目に遭う
5. **結び**: 全員で「やっぱり今の現実が一番マシかもね」と結論づける

### セクション設計のポイント

- **推奨セクション数**: 5セクション（上記の5パートに対応）
- **合計セリフ数**: 60-120程度
- **重要**: 各パートの役割を明確にし、思考実験の流れを意識する

### 各セクションの推奨セリフ数

- 導入（妄想）：10-20
- 定義（開始）：10-20
- 展開（検証）：25-40
- オチ（悲劇）：15-25
- 結び：8-15

## セクション定義

各セクションには以下の情報を定義してください：

- **section_key**: 英語の識別子（例：introduction, definition, development, punchline, conclusion）
- **section_name**: 日本語の表示名（例：「導入（妄想）」「定義（開始）」「展開（検証）」「オチ（悲劇）」「結び」）
- **purpose**: セクションの目的（1-2文）
- **content_summary**: 具体的な内容（2-3文）。**テーマと思考実験の流れを必ず含める**
- **min_lines / max_lines**: セリフ数の範囲（5-40）
- **background**: 背景シーン（形式：`{場所}_{詳細}_{状態}`、例：home_livingroom_bright）
  - **絶対厳守**: 全セクションで必ず同じ背景を指定してください。動画全体を通して一貫した場所で展開させます。場所を変えてはいけません。
