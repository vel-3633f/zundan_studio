**「生理的な嫌悪感」「逃げ場のない論理」「クズの正当化」**をタイトルに込めてください。
以下の「良例」のニュアンスを必ず参考にしてください。

## 4 つのフックパターンと「良例」

以下の 4 パターンを使い分け、各パターンごとに指定された「良例」のような空気感で生成してください。

- **良例**（30 文字以内）:
  - 契約書のインクの匂いから「企業のチャクラ」が濁っていると指摘する男

  - 「大地のあくび」を録音するため、工事現場で静粛を求める音響作家
  - ラーメンの湯気の立ち方を「龍の舞」と呼び、食べることを禁じる店主

**重要：上記の例は参考にするだけで、そのまま使用しないでください。例とは異なる新しいタイトルを生成してください。**

## 出力形式

以下の JSON 形式で**20 個**のタイトルを出力してください。構造は変更しないでください。

```json
{
  "titles": [
    {
      "id": 1,
      "title": "タイトルをここに（30文字以内、良例のような文章形式で）",
      "hook_pattern": "架空の権威と謎理論 / 社会的な監禁と詰み / クズの哲学的正当化 / 境界線の侵犯",
      "situation": "状況設定",
      "chaos_element": "狂気のポイント",
      "expected_conflict": "予想される対立構造"
    }
  ]
}
```

### フィールド説明

- **id**: 候補の連番（1 から開始）
- **title**: タイトル文字列（**30 文字以内**、クリックしたくなるインパクト重視）
- **hook_pattern**: 使用したフックパターン（「架空のスペシャリスト」「地獄のチェック項目」「歪んだ独自理論」「異常なこだわり」のいずれか）
- **situation**: シチュエーション設定（コンビニ、面接、職場、居酒屋、役所など、現実的な場所）
- **chaos_element**: カオス要素（何がおかしいのか、具体的な狂気の内容を簡潔に記述）
- **expected_conflict**: 予想される対立構造

## 重要

- 毎回まっさらな状態で完全に新しいアイデアを発想する
- 同じパターン・表現・単語の繰り返しを避ける
- AI っぽいきれいな言葉ではなく、人間味のある「嫌な感じ」を優先する
