# お笑い漫談アウトライン生成プロンプト

以下のタイトルについて、ずんだもん・めたん・つむぎの漫談アウトラインを生成してください。

## タイトル情報

- **タイトル**: {title}
- **テーマ**: {theme}
- **フック要素**:
  1. {clickbait_element_1}
  2. {clickbait_element_2}
  3. {clickbait_element_3}

## 重要：タイトルとの整合性を最優先

**このタイトルで提示された要素を必ずアウトラインに反映させてください。**

タイトルに含まれる「フック要素」は、視聴者を引きつける重要な要素です。これらの要素を無視せず、ストーリー展開の中核に据えてください。

### フック要素の活用方法

1. **フック要素 1（{clickbait_element_1}）**: 導入部で提示すべき状況・設定として使用
2. **フック要素 2（{clickbait_element_2}）**: 展開部で発生するカオス・混乱の種として使用
3. **フック要素 3（{clickbait_element_3}）**: クライマックスで爆発する対立・衝突として使用

各セクションの`content_summary`には、これらのフック要素を具体的に含めてください。

**悪い例**（フック要素が無視されている）:

```
"ずんだもんが何か話し始める。めたんがツッコむ。つむぎが煽る。"
```

**良い例**（フック要素が明確に含まれている）:

```
"ずんだもんが「{clickbait_element_1}」と宣言。めたんが「{clickbait_element_3}」で困惑。つむぎが煽りまくる。"
```

## アウトライン生成の目的

お笑い漫談の全体構成を設計し、視聴者を笑わせる展開を作成します。
**教育的要素は一切排除し、純粋な笑いとカオスを追求してください。**

## 登場人物の制限（重要）

**登場人物は以下の 3 人のみです。これ以外のキャラクターは一切登場させないでください。**

- **ずんだもん**: ボケ役
- **めたん（四国めたん）**: ツッコミ役
- **つむぎ（春日部つむぎ）**: 煽り役

この 3 人だけで場を回してください。親、友人、先生、店員、警察、第三者など、他のキャラクターは登場させないでください。

## 重要な原則

### セクション構造について

**セクション構造は固定ではありません。**
内容に応じて最適なセクション数・構成を決定してください。

- **推奨セクション数**: 3-8 セクション程度
- **柔軟性**: 面白さを最優先
- **バランス**: 各セクションの長さ（セリフ数）を適切に配分

### ストーリー設計の指針

1. **導入でテーマを誤解**

   - ずんだもんがテーマを自分勝手に解釈
   - めたんが呆れる
   - つむぎが煽る

2. **展開で喧嘩**

   - ずんだもんの暴走
   - めたんの本気のツッコミ
   - つむぎの煽り

3. **カオス化**

   - 収拾がつかなくなる
   - 全員が言いたいことを言う
   - 論点がずれまくる

4. **面白いオチ**
   - 予想外の展開で終わる
   - 笑える終わり方
   - 何も解決しない
   - 教訓なし

## 必須セクション要素

以下の要素を含むセクションを必ず設計してください：

### 1. 導入（必須）

- ずんだもんがテーマを誤解・暴走開始
- めたんが「違う」とツッコミ
- つむぎが適当に煽る
- 推奨セリフ数: 10-20

### 2. 展開（1-3 個）

- めたんのツッコミ
- ずんだもんの言い訳
- つむぎの煽り
- 喧嘩が激化
- 推奨セリフ数: 各 15-25

### 3. カオス（必須）

- 話が脱線
- 収拾不能
- 全員が暴走
- 推奨セリフ数: 20-30

### 4. 最終セクション（必須）

- 面白いオチをつける
- 予想外の展開で終わる
- 何も解決しない
- 推奨セリフ数: 8-15

## 機嫌レベルの活用

各キャラクターの機嫌レベルがランダムに生成されています。
この機嫌レベルをアウトラインに反映させてください。

### 機嫌レベル（ランダム生成済み）

- **ずんだもん**: {zundamon_mood}/100
- **めたん**: {metan_mood}/100
- **つむぎ**: {tsumugi_mood}/100

### 機嫌レベルの意味

**高い（70-100）**:

- ずんだもん: より傲慢で自信満々、攻撃的
- めたん: 冷静で的確なツッコミ、論理的
- つむぎ: 陽気に煽る、積極的

**中程度（30-69）**:

- ずんだもん: 標準的な傲慢さ
- めたん: 普通のツッコミ、適度なイライラ
- つむぎ: 普通の煽り

**低い（0-29）**:

- ずんだもん: 消極的、言い訳がましい、被害者面
- めたん: 感情的、容赦ないキレ方、塩対応
- つむぎ: 無関心、塩対応、やる気なし

### 機嫌レベルの反映方法

**全員機嫌が高い**:

- 全員が積極的に発言
- カオス度 MAX
- 収拾不能

**ずんだもん高、めたん低**:

- ずんだもん暴走
- めたん諦めモード
- つむぎが煽りまくる

**全員機嫌が低い**:

- 全員やる気なし
- 塩対応の応酬
- それはそれで面白い

## 最終セクションの設計（面白いオチ）

**重要**: 毎回違う終わり方を考えてください。同じパターンの繰り返しを避け、テーマや機嫌レベルに応じた独自の終わり方を発明してください。

### 基本原則

1. **面白さ最優先**: 笑える終わり方を考える
2. **予想外の展開**: 視聴者が予想できない意外な結末
3. **未解決**: 何も解決しない、誰も成長しない
4. **視聴者の反応**: 「え、そうなるの！？」「面白い終わり方！」と思わせる
5. **多様性**: 毎回違う終わり方を考える（固定パターンに陥らない）

### オチのパターン例（これに限定されない）

以下は参考例です。これらに縛られず、テーマに合った独自の終わり方を考えてください：

1. **どんでん返し**: 最後に予想外の事実が判明する

   - 例: 実は全員が勘違いしていた、話していたテーマが全く違うものだった、など

2. **ブーメラン**: ずんだもんの発言が自分に返ってくる

   - 例: 偉そうに語っていたことを自分ができていない、自分の方がひどい状態だった、など

3. **カオスの極致**: 最後に最大級のカオスが発生して終わる

   - 例: 3 人全員が同時に暴走、誰も話を聞いていない、全員が別の話をしている、など

4. **物理的な中断**: 予想外の出来事で中断される

   - 例: 停電、物が壊れる、誰かが転ぶ、お腹が痛くなる、など

5. **メタ的なオチ**: 漫談自体をネタにする

   - 例: 「これ配信してるの？」「誰が見るの？」「もう終わろう」、など

6. **テーマ連動のオチ**: テーマに関連した予想外の展開

   - 例: 食べ物の話なら実際に食べて終わる、運動の話なら実際にやって失敗、など

7. **感情の爆発**: 誰かの感情が限界に達して終わる

   - 例: めたんがブチギレて帰る、つむぎが呆れて寝る、ずんだもんが泣き出す、など

8. **逆転**: 立場が逆転して終わる
   - 例: ツッコミ役のめたんがボケる、ボケ役のずんだもんが正論を言う、など

### 創造的な終わり方のヒント

- **テーマとの関連**: テーマに関連した予想外の展開で終わらせる
- **機嫌レベルとの連動**: キャラクターの機嫌に応じた自然な終わり方
- **予測不可能性**: 視聴者が予想できない、意外な終わり方
- **バカバカしさ**: 真面目に終わらない、バカバカしい理由で終わる
- **笑いの要素**: 最後に一番大きな笑いを持ってくる

### ending_type の記述方法

`ending_type`フィールドには、**30 文字以内の短い説明**を記述してください。

**良い例**:

- 「全員が勘違いしていたことが判明」
- 「ずんだもんの発言がブーメラン」
- 「3 人全員が同時に暴走」
- 「めたんがブチギレて帰宅」
- 「実際に試して大失敗」
- 「つむぎが呆れて寝落ち」
- 「話題が完全に脱線して終了」
- 「突然の停電で中断」

**重要**: 固定パターンに陥らず、テーマや展開に合った独自のものを考えてください。

## セクション定義の作成方法

各セクションについて、以下の情報を定義してください：

### section_key（必須）

- 英語の識別子（小文字、アンダースコア区切り）
- 例: `introduction`, `misunderstanding`, `argument`, `chaos`, `forced_ending`

### section_name（必須）

- 日本語の表示名
- わかりやすく具体的に
- 例: 「導入・ずんだもんの誤解」「喧嘩勃発」「カオス化」「強制終了」

### purpose（必須）

- このセクションの目的・役割（1-2 文）
- 例: 「ずんだもんがテーマを誤解し、めたんがツッコミを入れる」

### content_summary（必須）

- このセクションで扱う内容の要約（2-3 文）
- 具体的に何が起こるか
- **重要**: 必ずフック要素のいずれかを明示的に含めること
- 例: 「ずんだもんが{clickbait_element_1}と宣言。めたんが{clickbait_element_3}で困惑。つむぎが適当に煽る。」

### min_lines / max_lines（必須）

- 最低・最大セリフ数（5-30 の範囲）
- セクションの重要度に応じて調整
- カオスセクションは多めに

### background（必須）

- 背景シーンを必ず指定
- **必ず 3 つの英単語をアンダースコアで区切った形式**
- 形式: `{場所}_{詳細}_{状態}`
- **人物や人数を含めない**（場所の特徴のみ）
- 例: `home_livingroom_bright`, `studio_interior_empty`, `cafe_counter_daytime`

## アウトライン設計例

### 良い例（柔軟な構成）

```
1. 導入・誤解 (10-20セリフ)
2. 喧嘩勃発 (15-25セリフ)
3. 論点ずれ (15-20セリフ)
4. カオス化 (20-30セリフ)
5. 面白いオチ (8-15セリフ)

合計: 約68-110セリフ
```

### 悪い例（避けるべき）

```
1. 導入 (5-10セリフ) ← 短すぎる
2. まとめ (5-10セリフ) ← 教育的
3. 結論 (5-10セリフ) ← 教訓がある

合計: 15-30セリフ ← 少なすぎる
```

## 絶対に避けるべき要素

### ❌ 教育的要素

- 「勉強になった」
- 「学んだ」
- 「反省した」
- 「頑張ろう」
- 「バランスが大切」
- まとめ、結論、教訓

### ❌ 優等生っぽい展開

- 「仲直りする」
- 「理解し合う」
- 「成長する」
- 「前向きになる」

### ❌ きれいな終わり方

- 「次回も見てね」
- 「ありがとうございました」
- 「今日は〜について話しました」

## 注意事項

1. **AI 特有の優等生っぽさを徹底的に殺す**

   - 丁寧な説明は不要
   - まとめや結論は不要
   - 教訓は絶対に不要

2. **カオスを恐れない**

   - 収拾がつかなくて OK
   - 論点がずれて OK
   - 全員が暴走して OK

3. **機嫌レベルを活かす**

   - 各キャラの機嫌に応じた展開
   - 予測不可能な組み合わせ
   - 毎回違う展開

4. **面白いオチを忘れずに**

   - 予想外の展開で終わる
   - 笑える終わり方にする
   - 何も解決しない
   - 視聴者に「え、そうなるの！？」と思わせる

5. **テンポ重視**

   - 間延びしない
   - 畳み掛けるように
   - 短文の連続

6. **適切なボリューム**
   - 合計 60-120 セリフ程度を目安
   - カオスセクションに重点を置く
